## Findings

### F-01 βÄî Security / Input-Validation βÄî MEDIUM
**Datei + Zeile:** `backend/api/external_v1.py:149`, `backend/api/performance.py:31`

`benchmark`-Query-Parameter hat keine Regex-Validierung in beiden Endpoints (`GET /api/v1/external/performance/history` und dem internen `GET /api/performance/history`). Ein beliebiger String wird ohne Sanitierung als yfinance-Ticker an `get_portfolio_history` weitergegeben, der ihn in einen `yf_download`-Call in einem Thread-Pool einhΟΛngt. Kein Code-Injection-Risiko (der String geht nur an yfinance), aber DoS-Vektor: ein sehr langer String oder ein String mit Sonderzeichen kΟΕnnte yfinance-Timeouts und blockierte Thread-Pool-Slots verursachen.

**Empfehlung:** In beiden Endpoints ersetzen:
```python
benchmark: str = Query(default="^GSPC")
```
durch:
```python
benchmark: str = Query(default="^GSPC", pattern=r"^[\^A-Z0-9.\-=]{1,20}$")
```

**Datei + Zeile:** `backend/api/external_v1.py:342`, `backend/api/external_v1.py:357`

### F-05 βÄî Architektur / Timezone-FragilitΟΛt βÄî INFO
**Datei + Zeile:** `backend/api/external_v1.py:600`


Macht `datetime.now(timezone.utc)` timezone-naiv durch `.replace(tzinfo=None)`, um ihn mit dem timezone-naiven `scan.started_at` (PostgreSQL `DateTime` ohne tz) vergleichen zu kΟΕnnen.
