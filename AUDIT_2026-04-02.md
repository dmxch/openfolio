## Detailbefunde

### PERF-H1: Fehlender Composite-Index auf `positions`

Die `positions`-Tabelle hat in der Live-DB nur 3 Indizes:
- `positions_pkey` (id)
- `uq_position_user_ticker` (user_id, ticker)
- `ix_positions_user_id` (user_id)

Das SQLAlchemy-Model definiert `Index("ix_positions_user_active", "user_id", "is_active")` in `/home/harry/projects/openfolio/backend/models/position.py:52`, und Migration 023 erstellt einen Index namens `idx_positions_user_active` — aber NUR auf `["user_id"]`, nicht als Composite. Der Model-Name und der Migrations-Name weichen ab, der Composite wurde nie tatsächlich erstellt.

Der Worker feuert alle 60s Queries mit `WHERE is_active = TRUE` — ohne Composite-Index führt PostgreSQL bei 31 Rows einen Seq-Scan durch (Overhead akzeptabel bei 31 Rows, aber Muster wird bei mehr Usern zum Problem).

Die 271 471 Seq-Scans auf `positions` sind auf die Worker-Frequenz zurückzuführen (ca. 60s Intervall × 4 500 Stunden Laufzeit).

### PERF-M1: Sync HTTP-Calls in Event-Loop (portfolio_service)

In `/home/harry/projects/openfolio/backend/services/portfolio_service.py`:

```python
# Zeile 170 — direkt im async Body von get_portfolio_summary()
market_value_chf, current_price, price_currency, stale_info = _compute_market_value(pos, fx_rates)
```

`_compute_market_value` ruft `get_stock_price()` auf, welche `yf.Ticker(ticker).fast_info` ausführt. Der `_in_event_loop()`-Guard in `price_service.py:15` gibt bei aktivem Event-Loop `None` zurück — daher kein Blockieren im Happy Path, aber ein stiller Fehler wenn der Cache kalt ist.

### QA-H1: Test-Suite nicht ausführbar

```
ImportError: No module named 'sqlalchemy'
tests/conftest.py:16: in <module>
    from sqlalchemy import create_engine
```

Die lokale Python-Umgebung hat keine Backend-Abhängigkeiten installiert. `pytest` kann ausserhalb des Containers nicht ausgeführt werden. Ein CI-Step der Tests im Docker-Build-Kontext ausführt fehlt.

### ARCH-M1: FastAPI-Import in Service-Layer

`/home/harry/projects/openfolio/backend/services/admin_service.py:9`:
```python
from fastapi import HTTPException, Request
```
`Request` wird als Parameter für `log_admin_action` übergeben — die IP-Adresse wird aus dem Request extrahiert. Dies koppelt den Service an FastAPI. Alternativer Ansatz: IP als `str`-Parameter übergeben, Extraktion im Router.

### ARCH-L2: Migrations-Drift (Migration 038 nicht angewendet)

DB-Version ist `037`, aktuellste Migrationsdatei ist `038_add_margin_rate_to_mortgages.py`. Die `mortgages`-Tabelle fehlt das `margin_rate`-Feld in der Live-DB. Betrifft Real-Estate-Funktionalität.

## Staerken

**Security:** Hervorragend. Rate-Limiting auf allen Auth-Endpoints (18 Limits), vollstaendige Pydantic-Validierung aller Input-Schemas, AES-256 Verschluesselung fuer PII-Felder (Mieter, IBAN, Adresse), korrekte CORS-Konfiguration mit Wildcard-Schutz, JWT-Geheimnis- und Encryption-Key-Validierung beim Start, `no-new-privileges` + `cap_drop: ALL` auf allen Containern. Lodash-Vulnerability ist ausschliesslich transitiv ohne exploitbaren Pfad im App-Code.

**Docker/12-Factor:** Alle Secrets via Umgebungsvariablen, keine Hardcoded-Credentials in `docker-compose.yml`, Memory- und CPU-Limits auf allen Services, Health-Checks auf allen Containern, Ports nur auf `127.0.0.1` gebunden.

**Nginx:** CSP, HSTS, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy korrekt gesetzt. Rate-Limiting am Nginx-Layer (`60r/m`). `server_tokens off`. Proxy-Timeouts konfiguriert.

**SMTP:** Korrektes `aiosmtplib` (nicht `smtplib`) in allen 3 Service-Dateien.

**yfinance-Wrapper:** `yf_download()` mit `asyncio.to_thread()` korrekt implementiert. Score-Endpoint, MRS-History, Breakouts alle via `to_thread` gewrapped.

**Accessibility:** `useFocusTrap` und `useEscClose` auf allen modalen Dialogen. `role="dialog"`, `aria-modal="true"`, `aria-label` auf allen Overlays. `ErrorBoundary` mit Auto-Reload bei Chunk-Fehlern und Fehler-Reporting ans Backend. 166 ARIA-Attribute im Frontend.

**DB-Schema:** Kaskadierendes `ondelete="CASCADE"` auf allen User-FK-Beziehungen. Transactions-Tabelle hat 10 optimierte Indizes.

**Error Messages:** Konsequent auf Deutsch fuer User-facing Responses, Englisch in Logs — durchgaengig eingehalten.

**Monitoring:** Prometheus + Loki + Grafana + Uptime-Kuma vollstaendig konfiguriert, alle Ports auf `127.0.0.1`, Grafana anon-auth deaktiviert.
